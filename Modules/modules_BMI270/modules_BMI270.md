# 飞控 IMU 坐标系与数据标定规范

本文档定义了本飞控框架（基于 MicoAir743 V2 等硬件）的惯性测量单元（IMU）坐标系标准，以及在不同物理状态下，底层驱动向上层算法输出的预期数据特征。

## 1. 物理坐标系基准：FRD (前-右-下)

本系统采用航模与开源飞控中最标准的 **FRD (Forward-Right-Down)** 右手坐标系。

* **X 轴 (Forward / 前)**：顺着飞控板面上的白色三角形丝印箭头，指向机头正前方。
* **Y 轴 (Right / 右)**：指向机头正前方的右侧。
* **Z 轴 (Down / 下)**：垂直于飞控板面向下（指向地心）。

---

## 2. 统一物理量单位

在 `BMI088_GetData` 或 `BMI270_GetData` 输出的 `BMI088_Data_t` 结构体中，物理量必须严格统一为以下国际标准单位（SI），以便无缝对接 EKF / Mahony 姿态解算算法：

* **加速度 (Accel)**：`m/s²` (米/平方秒)。标准重力加速度 $g \approx 9.80665$ `m/s²`。
* **角速度 (Gyro)**：`rad/s` (弧度/秒)。注意不是 `deg/s` (度/秒)。
* **时间戳 (Timestamp)**：`us` (微秒)，使用 64 位无符号整型保证不溢出。

---

## 3. 加速度计 (Accelerometer) 预期数据

**核心物理准则**：加速度计测量的**不是重力**，而是**抵抗重力的支撑力**。因此，静止状态下，传感器输出的是与其所受支撑力方向一致的加速度值。

### 3.1 静态姿态校验表 (极其重要)

| 测试动作 (保持绝对静止) | 物理受力分析 (支撑力方向) | accel[0] (X轴) | accel[1] (Y轴) | accel[2] (Z轴) |
| :--- | :--- | :--- | :--- | :--- |
| **水平放置** (正常平飞) | 桌子向上托起飞控 (-Z 方向) | ~ 0 | ~ 0 | **-9.8** |
| **倒扣放置** (背面朝上) | 桌子向下托起飞控 (+Z 方向) | ~ 0 | ~ 0 | **+9.8** |
| **机头朝下** (垂直立起) | 桌子向后托起飞控 (-X 方向) | **-9.8** | ~ 0 | ~ 0 |
| **机头朝上** (垂直立起) | 桌子向前托起飞控 (+X 方向) | **+9.8** | ~ 0 | ~ 0 |
| **右侧朝下** (右翻转 90°) | 桌子向左托起飞控 (-Y 方向) | ~ 0 | **-9.8** | ~ 0 |
| **左侧朝下** (左翻转 90°) | 桌子向右托起飞控 (+Y 方向) | ~ 0 | **+9.8** | ~ 0 |

> **注**：部分算法库（如某些特定的 Mahony 实现）可能在数学定义上期望平放时 Z 轴输出 `+1g`（即 +9.8）。如果遇到解算上下颠倒，请优先检查算法库对静态重力向量方向的约定，必要时在驱动解析层对特定轴取反。

### 3.2 动态平移校验表

| 测试动作 (突然加速推) | accel[0] (X轴) | accel[1] (Y轴) | accel[2] (Z轴) |
| :--- | :--- | :--- | :--- |
| **向正前方猛推** | **正向尖峰 (>0)** | 无明显变化 | 保持 -9.8 附近 |
| **向正右方猛推** | 无明显变化 | **正向尖峰 (>0)** | 保持 -9.8 附近 |
| **向正下方猛按** | 无明显变化 | 无明显变化 | **正向尖峰 (>0)** |

## 4. 陀螺仪 (Gyroscope) 预期数据

**核心物理准则**：严格遵循**右手螺旋定则**。大拇指指向某坐标轴的正向，四指弯曲的方向即为该轴的正向旋转（角速度 > 0）。

### 4.1 动态旋转校验表

| 测试动作 (握住板子快速转动) | 航空术语 | gyro[0] (X轴) | gyro[1] (Y轴) | gyro[2] (Z轴) |
| :--- | :--- | :--- | :--- | :--- |
| **右侧机翼下压** (顺时针滚转) | Roll Right | **正向输出 (>0)** | ~ 0 | ~ 0 |
| **左侧机翼下压** (逆时针滚转) | Roll Left | **负向输出 (<0)** | ~ 0 | ~ 0 |
| **抬起机头** (仰角增大) | Pitch Up | ~ 0 | **正向输出 (>0)** | ~ 0 |
| **压低机头** (俯角增大) | Pitch Down | ~ 0 | **负向输出 (<0)** | ~ 0 |
| **机头向右平转** (顺时针偏航) | Yaw Right | ~ 0 | ~ 0 | **正向输出 (>0)** |
| **机头向左平转** (逆时针偏航) | Yaw Left | ~ 0 | ~ 0 | **负向输出 (<0)** |

---

## 5. 驱动层映射规范

由于传感器芯片 (BMI088 / BMI270) 在 PCB 上的贴片方向（旋转、翻面）可能与板子的 FRD 丝印不一致，驱动层有责任在 `ParseData` 阶段完成坐标系的重映射。

**原则**：暴露给算法层的 `accel[3]` 和 `gyro[3]` 数组，必须**无条件**满足上述表格中的所有特征。如果发现不符，应在驱动层进行如下操作之一：
1.  **轴交换**（例如：芯片转了 90 度，导致芯片的 X 成了板子的 Y）。
2.  **符号取反**（例如：芯片反面贴片，导致某些轴刚好反向）。

确保算法层（EKF / PID）接收到的永远是绝对标准的 FRD 数据，从而实现算法与硬件的彻底解耦。
